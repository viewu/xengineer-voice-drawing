# AI 语音绘图工具设计文档

## 1. 题目理解

比赛要求开发一款纯语音控制的绘图工具，用户不能使用鼠标或键盘，仅通过语音指令完成绘图创作。评审重点包括指令理解准确性与容错性、语音到绘图操作的响应延迟，以及复杂指令拆解与执行能力。

## 2. 用户故事

### 计划实现

1. 作为用户，我可以说“画一个红色圆形”，系统立即在画布上添加圆形。
2. 作为用户，我可以说“画一个蓝色矩形，然后添加一条绿色箭头”，系统把一句话拆成多个绘图步骤。
3. 作为用户，我可以通过语音控制颜色、填充、粗细，让作品更接近表达意图。
4. 作为用户，我可以说“撤销”“重做”“清空画布”，修正创作过程中的错误。
5. 作为用户，我可以说“向右移动最后一个图形”“放大最后一个图形”，调整最近生成的元素。
6. 作为用户，我可以导出当前画布，用于保存或分享。

### 最终实现

已实现上述 1-6 项。当前版本重点支持常见图形、基础样式和最后元素编辑，适合作为比赛 MVP 展示。

## 3. 指令能力

### 图形创建

- 矩形：`画一个蓝色矩形`
- 圆形：`画一个红色实心圆形`
- 线条：`画一个粗的紫色线条`
- 箭头：`添加一条绿色箭头`
- 星星：`画一个黄色星星`
- 文字：`写上 Hello`

### 画布操作

- 撤销：`撤销`
- 重做：`重做`
- 清空：`清空画布`
- 导出：`导出`

### 编辑操作

- 移动最后一个图形：`向右移动最后一个图形`
- 放大最后一个图形：`放大最后一个图形`
- 缩小最后一个图形：`缩小最后一个图形`

## 4. 架构设计

项目采用无构建的静态 Web 架构，降低部署和演示成本。

```text
语音输入
  ↓
Web Speech API
  ↓
commandParser.js
  ↓
drawingState.js
  ↓
Canvas 2D 渲染
  ↓
语音反馈 / 指令历史 / PNG 导出
```

### 模块职责

- `src/commandParser.js`：把自然语言文本解析为标准动作对象。
- `src/drawingState.js`：维护图形列表、撤销栈、重做栈和最后元素调整。
- `src/app.js`：连接语音识别、状态执行、Canvas 绘制、语音反馈和导出。
- `tests/`：验证核心解析和状态行为。

## 5. 复杂指令拆解

解析器会识别以下连接词，并把一句话拆成多个子指令：

- `然后`
- `再`
- `接着`
- `并且`
- `and`
- `then`

示例：

```text
画一个蓝色矩形，然后添加一条绿色箭头
```

会被拆为：

1. 添加蓝色矩形
2. 添加绿色箭头

## 6. 容错策略

当系统无法解析指令时，不修改当前画布，并给出可执行示例：

```text
没有理解这条指令。你可以说：画一个红色圆形，然后添加一条蓝色箭头。
```

这样可以减少用户困惑，也方便 demo 时快速恢复。

## 7. 响应延迟与成本控制

当前版本的指令解析在浏览器本地完成，不把每一句语音都发送到云端模型。这样有三个好处：

1. 延迟低：语音识别完成后即可执行动作。
2. 成本低：常见指令不消耗模型调用费用。
3. 可控性强：比赛 demo 中行为稳定，不依赖外部 API 可用性。

未来可以采用分层策略：本地规则解析优先，只有无法理解的复杂表达才调用大模型进行语义解析。

## 8. 未完成部分与原因

- 暂未支持任意位置精确放置：语音位置表达差异较大，MVP 先使用自动布局，保证创作流畅。
- 暂未支持多图层管理：比赛 demo 优先展示语音到绘图闭环，图层系统可作为后续扩展。
- 暂未接入云端大模型：为了控制成本、降低延迟并保证离线可演示，当前版本采用本地解析。

## 9. 后续扩展
